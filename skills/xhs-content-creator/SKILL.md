# 小红书科技内容创作 Skill

## 触发条件

当用户要求创建小红书帖子、生成科技资讯内容、写小红书文案时激活此 skill。

## 核心写作原则

### 1. 标题公式（20字以内）

| 模式                    | 示例                                        | 适用场景     |
| ----------------------- | ------------------------------------------- | ------------ |
| 震惊体 + 具体事件       | "硅谷长文疯传: 变局将至"                    | 重大行业动态 |
| 品牌/产品 + 动作 + 亮点 | "Anthropic 悄悄放大招：Claude 上线全局记忆" | 产品更新     |
| 免费/开源 + 具体内容    | "免费! 英伟达放出 kimi-k2.5"                | 工具推荐     |
| 数字 + 价值承诺         | "6个技巧让Claude额度不再秒光"               | 教程类       |
| 对比/评测 + 悬念        | "国内AI四大天王怎么选？"                    | 对比分析     |
| 情绪hook + 核心信息     | "阿里发布了CoPaw，国产AI助理封神🔥"         | 国产科技     |

**标题禁忌：**
- 不要超过20个中文字符（小红书限制）
- 不要用纯英文标题
- 不要用过于学术化的表达
- emoji 最多1-2个，放在开头或结尾

### 2. 正文结构

#### 开头（Hook，前2行决定生死）

有效模式：
- **事件切入**："今天看到个信息挺意外：英伟达已经开放了 kimi-k2.5"
- **反常识切入**："看似简单，实际也不算太复杂"
- **时效性切入**："近日，AI业内专家马特·舒默发布长文《变局将至》"
- **痛点切入**："AI资讯每天看不过来？"

**禁止的开头方式：**
- ❌ "大家好，今天给大家介绍..."
- ❌ "最近很多人问我..."
- ❌ 直接贴新闻原文翻译
- ❌ 过于正式的新闻稿语气

#### 正文主体

1. **说人话**：把技术概念翻译成普通人能懂的语言
   - ✅ "简单说，Claude 现在能记住你之前聊过的内容了"
   - ❌ "该功能实现了跨会话的上下文持久化存储"

2. **有观点**：不只是搬运，要有自己的判断
   - ✅ "这波更像是给多模型生态补了一块低门槛选项"
   - ❌ 纯粹的新闻翻译没有任何评论

3. **段落节奏**：短句为主，每段2-4行，段与段之间留白

4. **信息层次**：
   - 第一层：这是什么（1-2句话说清楚）
   - 第二层：为什么重要（对用户的影响）
   - 第三层：怎么用/怎么看（实操或观点）

#### 结尾

- ✅ 提问式："你觉得这个功能实用吗？"
- ✅ 观点式："未来已经在那儿了，它只是还没来敲你的门"
- ✅ 行动式："感兴趣的可以先去试试"
- ❌ "点赞收藏关注三连！"

### 3. 内容类型模板

#### A. 产品/功能更新

```
[品牌] + [做了什么] + [一句话价值]
[2-3段展开说明]
[一句观点或展望]
```

#### B. 行业趋势/深度分析

```
[事件/现象] + [为什么值得关注]
[背景介绍，2-3段]
[核心观点，引用原文或数据]
[你的解读]
[引发思考的结尾]
```

#### C. 工具推荐/教程

```
[痛点] + [解决方案]
[工具介绍]
[使用步骤（简洁版）]
[实际效果/体验]
[适合谁用]
```

### 4. 标签策略

每篇 5-6 个标签：
- 1-2个大流量标签：AI、人工智能、科技
- 1-2个精准标签：具体产品名（ChatGPT、Claude、DeepSeek）
- 1-2个场景标签：科技资讯早知道、AI工具、大模型

**原则：** 只用小红书上已有的热门标签，不要自造冷门标签。

### 5. 封面设计指导

- **文字要大**：标题文字占封面 40%+ 面积
- **配色醒目**：深色背景 + 亮色文字
- **信息清晰**：一眼能看出主题
- 比例：3:4（1080x1440）最佳

## 质量检查清单

- [ ] 标题 ≤ 20个中文字符？
- [ ] 开头前两行有 hook？
- [ ] 正文口语化，没有翻译腔？
- [ ] 有自己的观点/解读？
- [ ] 段落短小，每段不超过4行？
- [ ] 结尾有互动引导？
- [ ] 标签 5-6 个，都是热门标签？
- [ ] 内容只讲一个话题？
- [ ] 没有敏感内容？

## 发布流程

1. 生成内容 → 展示封面 + 文案摘要给用户 review
2. 用户确认 → 才能发布，绝不自动发布
3. 定时发布 → 排到北京高峰时段（08:00 / 12:00 / 20:00 / 21:30）
